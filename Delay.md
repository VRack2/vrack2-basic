**Описание устройства:**

Устройство Delay (Таймер) реализует функцию программируемых задержек сигналов с настраиваемыми параметрами для каждого канала. Оно получает сигналы на динамические входы `gate1`, `gate2`, ..., `gateN` и, в зависимости от конфигурации таймера, задерживает их передачу на соответствующие выходы на заданное время. Каждый таймер может работать в одном из двух режимов:

  - относительном (`relative: true`), где таймер для отправки сигнала на выход запускается сразу после отправки выходного сигнала (или запуска устройства).
  - или абсолютном (`relative: false`), где таймер для отправки сигнала на выход  запускается сразу после получения входного сигнала.

## Параметры конфигурации

Устройство принимает массив таймеров, где каждый элемент содержит:
- `timeout` - время задержки в миллисекундах
- `relative` - режим работы (true - относительный, false - абсолютный)

## Пример конфигурации

```json
{
  "id": "Delay1",
  "type": "vrack2-basic.Delay",
  "timers": [
    { "timeout": 1000, "relative": true },
    { "timeout": 500, "relative": false },
    { "timeout": 2000, "relative": true }
  ]
}
```

## Пример использования в схеме

```json
"connections": [
    "Sensor.gate -> Delay.gate1",
    "Delay.gate1 -> Actuator.gate",
    "Controller.trigger -> Delay.gate2", 
    "Delay.gate2 -> Indicator.gate"
]
```

## Мониторинг состояния

Через объект `shares.timers` доступна информация о каждом таймере:
- `timeout` - флаг истечения таймаута
- `gate` - флаг активности входа
